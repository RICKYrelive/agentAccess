# LLM聊天功能设置指南

## 🎉 功能概述

现在Agent Access已经集成了真实的LLM聊天功能！你可以配置OpenAI API或其他兼容的模型提供商，然后与AI进行实时对话。

## 📋 功能特性

✅ **模型提供商配置**：支持OpenAI、Anthropic、Ollama等
✅ **实时流式对话**：支持流式响应，实时显示AI回复
✅ **多模型支持**：可配置多个模型提供商，自由切换
✅ **连接测试**：测试模型提供商的连接状态
✅ **聊天历史**：保存对话历史，支持清空对话
✅ **团队Agent**：智能数据分析、写作助手、股市分析等

## 🚀 快速开始

### 1. 配置模型提供商

1. 点击右侧设置按钮或"MCP服务设置"选项卡
2. 切换到"模型提供商"选项卡
3. 点击"添加模型提供商"按钮
4. 填写配置信息：
   - **名称**：例如"OpenAI GPT-4"
   - **类型**：选择"OpenAI"
   - **Base URL**：`https://api.openai.com/v1`（或其他兼容地址）
   - **API Key**：你的OpenAI API密钥
   - **模型**：`gpt-4`、`gpt-3.5-turbo`等
   - **最大令牌**：4000（可选）
   - **温度**：0.7（可选）

### 2. 测试连接

配置完成后，点击提供商卡片上的⚡按钮测试连接，确保配置正确。

### 3. 开始聊天

1. 回到主页
2. 可以选择团队Agent（智能数据分析、写作助手等）
3. 在中央大聊天框中输入你的问题
4. 点击发送按钮或按Ctrl+Enter发送消息
5. 实时查看AI回复

## 🔧 高级配置

### 支持的模型提供商类型

- **OpenAI**：官方OpenAI API
- **Anthropic**：Claude API
- **Ollama**：本地模型服务
- **自定义**：任何OpenAI兼容的API

### 使用其他OpenAI兼容服务

你可以使用任何OpenAI兼容的API服务，例如：

- **DeepSeek**：Base URL: `https://api.deepseek.com/v1`
- **智谱AI**：Base URL: `https://open.bigmodel.cn/api/paas/v4`
- **月之暗面**：Base URL: `https://api.moonshot.cn/v1`
- **本地模型**：使用Ollama等本地服务

### 配置示例

```
名称: DeepSeek V3
类型: OpenAI
Base URL: https://api.deepseek.com/v1
API Key: sk-your-deepseek-key
模型: deepseek-chat
最大令牌: 8000
温度: 0.7
```

## 💡 使用技巧

### 1. 选择合适的Agent

- **智能数据分析**：适合数据分析、图表解读、趋势分析
- **写作助手**：适合文案创作、内容优化、文档编写
- **股市基本面分析**：适合股票分析、投资建议、市场解读

### 2. 调整参数

- **温度**：控制回复的创造性
  - 0.1-0.3：更准确、更保守
  - 0.7-1.0：更有创造性、更随机
- **最大令牌**：控制回复长度
  - 短回答：1000-2000
  - 长回答：4000-8000

### 3. 优化对话效果

- 明确描述你的需求
- 提供足够的上下文信息
- 可以要求AI以特定格式回复
- 使用"请详细说明"、"请举例"等提示获得更好的回答

## 🛠️ 故障排除

### 连接失败

1. 检查API Key是否正确
2. 确认Base URL格式正确
3. 验证模型名称是否有效
4. 检查网络连接和防火墙设置

### 无响应

1. 确认模型提供商已激活
2. 检查API余额是否充足
3. 尝试测试连接功能

### 错误消息

如果看到错误消息，通常是因为：
- API Key无效
- 网络连接问题
- 模型服务不可用
- 请求参数错误

## 🔒 隐私说明

- 所有配置信息都保存在本地浏览器中
- API请求直接发送到你配置的提供商
- 不会将你的数据发送到第三方服务器
- 可以随时清除配置和对话历史

## 📞 技术支持

如果遇到问题：

1. 检查浏览器控制台的错误信息
2. 确认API配置正确
3. 尝试不同的模型提供商
4. 查看网络请求是否正常
